# 数据导入模板填写说明

**文档版本**: v1.1 · 2026-06-12  
**适用对象**: HR 人员  
**导入方式**: 在 ONA 平台点击「批量导入员工数据」，上传填写好的 .xlsx 或 .csv 文件

---

## 模板文件

模板文件位于平台前端页面 → 点击「批量导入员工数据」→ 点击「下载导入模板」按钮。

### 模板结构（双行表头）

Excel 文件第 1 行为中文引导行，第 2 行为英文列名（供系统解析），从第 3 行开始填入您的数据。

```
┌────────┬────────┬────────┬────────┬──────┬──────────────┬──────┬──────┐
│ 年龄   │ 状态   │ 出差频率│ 部门   │ 学历 │ 专业         │ 性别 │ 职级 │  ← 中文引导(跳过)
├────────┼────────┼────────┼────────┼──────┼──────────────┼──────┼──────┤
│ Age    │Attrition│Bus...│Depart..│Educ..│EducationField│Gender│Job..│  ← 英文列名(解析起点)
├────────┼────────┼────────┼────────┼──────┼──────────────┼──────┼──────┤
│ 35     │ No     │ Tra... │ Res... │  4   │Computer Sc.. │ Male │  3   │  ← 您从这行开始填数据
├────────┼────────┼────────┼────────┼──────┼──────────────┼──────┼──────┤
│ 28     │ No     │ Tra... │ Sales  │  3   │ Marketing    │Female│  2   │
└────────┴────────┴────────┴────────┴──────┴──────────────┴──────┴──────┘
```

> **技术说明**: 后端解析时自动跳过第 1 行中文引导，从第 2 行英文列名开始解析。第 1 行仅供 HR 对照填写。

---

## 字段说明

共 18 个字段，全部必填。填写时参照下方的中英文对照表。

### 字段对填表

| 第1行中文引导 | 第2行英文列名 | 类型 | 示例值 | 可选值 / 备注 |
|-------------|-------------|------|--------|-------------|
| 年龄 | `Age` | 整数 | `35` | 18 – 70 |
| 状态 | `Attrition` | 枚举 | `No` | `No` 在岗 · `Yes` 已离职 |
| 出差频率 | `BusinessTravel` | 枚举 | `Travel_Frequently` | `Travel_Frequently` 频繁 · `Travel_Rarely` 偶尔 · `Non-Travel` 不出差 |
| 部门 | `Department` | 枚举/文本 | `Research & Development` | `Research & Development` → 数智人力事业部 · `Sales` → 数智营销事业部 · `Human Resources` → 人力资源中心；不在表中则原样保留 |
| 学历 | `Education` | 整数 | `4` | 1=高中 · 2=大专 · 3=本科 · 4=硕士 · 5=博士 |
| 专业 | `EducationField` | 文本 | `Computer Science` | 如 Computer Science, Marketing, 人力资源 等 |
| 性别 | `Gender` | 枚举 | `Male` | `Male` 男 · `Female` 女 |
| 职级 | `JobLevel` | 整数 | `3` | 1=P1–P3 · 2=P4–P5 · 3=P6–P7 · 4=P8–P9 · 5=P10+ |
| 岗位 | `JobRole` | 枚举/文本 | `Research Scientist` | `Research Scientist` → 高级算法工程师 · `Laboratory Technician` → 实验室技术员 · `Manager` → 高级产品经理 · `Sales Executive` → 高级销售顾问 · `Manufacturing Director` → 制造总监 · `Sales Representative` → 售前顾问 · `Research Director` → 技术总监 · `Human Resources` → HRBP · `Healthcare Representative` → 医疗行业顾问；不在表中则原样保留 |
| 婚姻 | `MaritalStatus` | 枚举 | `Married` | `Married` 已婚 · `Single` 未婚 · `Divorced` 离异 |
| 月薪(元) | `MonthlyIncome` | 整数 | `32000` | 税前月薪 |
| 常加班? | `OverTime` | 枚举 | `No` | `No` 不常加班 · `Yes` 经常加班 |
| 涨薪% | `PercentSalaryHike` | 整数 | `15` | 百分比，如 15 表示涨薪 15% |
| 绩效 | `PerformanceRating` | 整数 | `4` | 1=不及格 · 2=中等 · 3=良好 · 4=优秀 · 5=卓越 |
| 总工龄 | `TotalWorkingYears` | 小数 | `12` | 可带小数 |
| 本公司工龄 | `YearsAtCompany` | 小数 | `6` | 可带小数 |
| 现岗年限 | `YearsInCurrentRole` | 小数 | `4` | 可带小数 |
| 距晋升年数 | `YearsSinceLastPromotion` | 整数 | `3` | — |

## 注意事项

1. **第一行不要删除** — 第 1 行中文引导供对照，第 2 行英文列名供系统解析，两行都要保留
2. **从第 3 行开始填数据** — 第 1、2 行为表头行，不要改动
3. **必填** — 18 个字段不能有空值，否则该行会跳过
4. **编码** — .csv 文件请使用 UTF-8 编码保存
5. **重复导入** — 系统自动去重，重复上传的数据会覆盖更新

## 常见错误

| 报错信息 | 原因 | 解决 |
|---------|------|------|
| "缺少必填字段" | 第 2 行列名拼写错误 | 核对模板中的英文列名，完全一致 |
| "文件解析失败" | 非法格式或编码不对 | 另存为 UTF-8 编码的 .csv 或标准 .xlsx |
| "不支持的文件格式" | 上传了 .pdf/.txt 等 | 只能上传 .csv 或 .xlsx |
