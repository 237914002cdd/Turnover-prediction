# 用户画像与场景分析

**文档状态**: v1.0 · 2026-06-12  
**适用范围**: 产品设计、用例验证、销售/售前材料  
**核心角色**: HRBP（业务人力资源合作伙伴）、CHO（首席人力官）/ HRD（人力资源总监）

## 1. 角色一：HRBP（业务人力资源合作伙伴）

### 1.1 User Profile

| 属性 | 描述 |
|------|------|
| **Title** | HR Business Partner / 业务人力资源合作伙伴 |
| **汇报线** | 虚线汇报给业务线 VP，实线汇报给 HRD |
| **管辖范围** | 300–2,000 人的业务单元（如产研中心、销售大区） |
| **典型画像** | 5–8 年 HR 经验，懂业务术语但非技术背景，Excel 熟练，SQL 基础 |
| **KPI** | 核心人才流失率 ≤ 8%、关键岗位填补周期 ≤ 30 天、组织健康度评分 |
| **决策权** | 可提议调薪/晋升/人员优化，但需业务主管 + HRD 双签 |
| **焦虑指数** | 🔴 高——每年 Q3/Q4 绩效季压力最大 |

### 1.2 Daily Workflow

| 时段 | 活动 | 工具/触点 |
|------|------|----------|
| 09:00–10:00 | 查收离职预警邮件、审阅昨日离职面谈记录 | 邮箱、HRIS |
| 10:00–11:30 | 与业务主管 1:1，讨论团队人员动态 | 飞书/企业微信 |
| 11:30–12:00 | 处理调薪/转正/优化审批流 | OA 系统 |
| 14:00–15:30 | 与高危员工做"留任面谈"（Retention Interview） | 会议室 |
| 15:30–16:30 | 整理离职原因分析周报 | Excel / BI 看板 |
| 16:30–18:00 | 跟进招聘 Pipeline、协调 HC 预算 | ATS 系统 |

### 1.3 Key Pain Points

| # | 痛点 | 频率 | 严重度 | 现状 |
|---|------|------|--------|------|
| P1 | **离职面谈流于形式**——员工在面谈中通常给出"套话"理由（家庭原因/个人发展），真实原因要到离职后 1–3 个月才从同事口中获知 | 每周 | 🔴 高 | 面谈记录表 + 主观判断 |
| P2 | **事后补救流产**——HRBP 往往是"最后一个知道"——员工提离职时才介入，黄金挽留窗口已过 | 每月 | 🔴 高 | 无预警机制 |
| P3 | **调薪/晋升缺乏客观数据**——向业务主管申请调薪预算时，对方问"凭什么给他加？"——HRBP 拿不出量化证据 | 每季度 | 🟡 中 | 历年绩效表 + 市场对标报告 |
| P4 | **跨部门协作冲突不可见**——两个部门的矛盾通常由某几个关键人引发，但 HRBP 只能看到结果（争吵/离职），看不到原因（谁在传递冲突） | 每月 | 🟡 中 | 访谈 + 主观判断 |
| P5 | **年度人力预算博弈**——申请留任预算时 HQ 要求 ROI 测算，HRBP 只能拍脑袋估算替代成本 | 每年 | 🟡 中 | Excel 静态成本计算表 |

### 1.4 ONA Value Propositions

| 痛点 | ONA 解法 | 核心价值 |
|------|---------|---------|
| P1 面谈流于形式 | 面谈前调取该员工的 SHAP 归因面板（Top 3 离职因子），面谈时直接切入真实问题 | 从"你觉得怎么样"变成"数据显示你的加班超载已达 50h/月" |
| P2 事后补救 | 自动离职预警（风险等级 + 概率）在员工提离职前 30–60 天推送至工作台 | 从"事后补救"到"事前干预" |
| P3 缺乏客观数据 | ROI 沙盘滑块：调薪 X% → 离职概率降至 Y% → 净节约 ¥Z | 拿数字找业务主管审批，不再被问住 |
| P4 跨部门冲突 | ONA 拓扑图中高 Hub Centrality 节点可能正是冲突放大器的两端 | 可视化识别组织"冲突枢纽" |
| P5 预算博弈 | 沙盘输出 PDF 决策报告，包含精确的 Net Savings 与 ROI 测算 | HRBP 拥有 CFO 听得懂的财务语言 |

### 1.5 User Story Mapping：调薪说服战役

> **场景**: HRBP 张琳发现产研中心高级工程师申鹏程（P5，月薪 ¥13,000）在 ONA 拓扑图中 Eigenvector Centrality = 0.96，离职概率 82%，且与 12 名同事存在强协作依赖。

**故事流**:

```
周一 09:30
张琳登录 ONA 平台 → 主页预警概览显示"1 高危 1 中危"
↓
点击"高危" → 拓扑图自动聚焦申鹏程，高亮其一度邻居（12 人）
↓
悬停节点 → 右侧 Drill-Down 面板弹出：
  · 离职概率: 82%
  · Top 3 因子: 薪资增幅(16.3%) / 任现职年限(14.1%) / 加班超载(13.0%)
  · 级联风险: 12 人潜在连带流失
↓
切换到 ROI 沙盘选项卡 → 拖动调薪滑块至 10%
  · 实时显示：离职概率降至 ~20% | 净节约 ¥2,520
↓
点击"导出报告" → 生成带 SHAP 图表 + ROI 测算的 PDF 决策报告
↓
周三 14:00
张琳带着 PDF 走进产研 VP 办公室：
  "申鹏程不是普通 P5——他是团队的非正式技术中枢。
  花 ¥15,600 调薪，省 ¥20,000 的替换成本，净赚 ¥2,520。
  这是 ONA 分析和 ROI 测算，都在报告里。"
↓
产研 VP 签字 → HRD 审批 → 周五完成调薪 → 离职概率降至 ~20%
```

## 2. 角色二：CHO（首席人力官）/ HRD（人力资源总监）

### 2.1 User Profile

| 属性 | 描述 |
|------|------|
| **Title** | Chief Human Resources Officer / VP of HR / HR Director |
| **汇报线** | 直接汇报给 CEO，部分矩阵汇报给 Board |
| **管辖范围** | 全组织 5,000–50,000+ 人 |
| **典型画像** | 15+ 年 HR 经验，MBA 或组织心理学背景，关注战略而非执行 |
| **KPI** | 人力资本 ROI、关键人才保留率、组织健康度、Diversity & Inclusion 指标 |
| **决策权** | 审批中高管任命、年度调薪预算池、组织架构调整 |
| **焦虑指数** | 🟡 中高——董事会季的压力最大 |

### 2.2 Daily Workflow

| 时段 | 活动 | 工具/触点 |
|------|------|----------|
| 09:00–10:00 | 高管晨会：组织健康度 Briefing | 会议室 / 飞书会议 |
| 10:00–11:00 | 审阅 HR Dashboard：离职率、招聘 Pipeline、继任计划 | BI 看板（Tableau / PowerBI） |
| 11:00–12:00 | 与 HRBP 团队周会：重点风险组织复盘 | 飞书 |
| 14:00–15:30 | Board 汇报准备：人力资本数据包 | PPT / 文档 |
| 15:30–16:30 | 战略项目评审：组织变革 / 文化转型 / M&A 人才尽调 | 会议室 / 外部顾问 |
| 16:30–18:00 | 行业 Benchmark 对比、供应商选型评估 | Gartner / Forrester 报告 |

### 2.3 Key Pain Points

| # | 痛点 | 频率 | 严重度 | 现状 |
|---|------|------|--------|------|
| P1 | **传统问卷主观偏差大**——Glint/Culture Amp 的 Pulse Survey 响应率逐年衰减，而且"满意度高 ≠ 不会走" | 每季度 | 🔴 高 | 持续倾听平台（响应率 30–50%） |
| P2 | **无法向 CEO/CFO 证明 HR 的直接财务 ROI**——CEO 问"去年花了 500 万做员工体验项目，效果在哪？"，CHO 只能说"敬业度提升了 3 分" | 每季度 | 🔴 高 | 敬业度分数 / eNPS / 定性案例 |
| P3 | **组织盲区**——不知道谁是隐性信息枢纽。调整一个中层管理者，可能无意中破坏了核心信息传递路径 | 每月 | 🟡 中 | 组织架构图（正式关系），看不到非正式网络 |
| P4 | **级联风险不可量化**——一个关键人走了，影响多大？现在的工具回答不了"连锁反应"的问题 | 每季度 | 🟡 中 | 无 |
| P5 | **供应商选型困难**——Workday、Visier、Glint 各说各话，缺少一个统合视角评估人才风险管理能力 | 每年 | 🟡 中 | RFP + Demo 对比 |

### 2.4 ONA Value Propositions

| 痛点 | ONA 解法 | 核心价值 |
|------|---------|---------|
| P1 问卷主观偏差 | 被动 ONA 替代主动调查：不依赖员工自报，基于协作元数据计算离职概率 | 数据客观、零负担、不可造假 |
| P2 HR ROI 无法证明 | 高层看板首页：高风险人才总期望替换损失（如 ¥200,064）——将人才风险翻译为 CFO 立马能懂的财务语言 | 用 CFO 的语言说话，不是 HR 的语言 |
| P3 组织盲区 | 全局拓扑图谱 + 焦点模式：识别非正式权力中心（Eigenvector Centrality > 0.9 的节点） | 看到组织架构图上看不到的人 |
| P4 级联风险 | Cascade Failure 算法：当高危员工离职时，自动计算连带影响的节点集合与财务敞口 | 从"一个人走了"到"系统级风险" |
| P5 供应商选型 | ONA 竞品分析已覆盖 4 家巨头的差异化定位（见 competitive_analysis.md） | 清晰的品类定位——我们不是另一个问卷平台 |

### 2.5 User Story Mapping：董事会汇报战役

> **场景**: CHO 王总将在下周一 Board Meeting 上汇报 Q2 人才风险状况。以往他只汇报"离职率 8.2%"——这次他打算用 ONA 数据讲一个不同的故事。

**故事流**:

```
周五 14:00
王总登录 ONA 平台 → 全局拓扑图谱加载完成
↓
看板顶部显示：
  · 高风险人才总期望替换损失: ¥200,064
  · 预警: 1 高危 / 1 中危 / 2 全员风险
↓
王总点击"风险-绩效九宫格" → 看到申鹏程落在"高流失风险"象限
  ——一个 P5 基层员工，却在九宫格上亮红
↓
展开 Drill-Down：
  · 离职概率: 82%
  · ON A Eigenvector Centrality: 0.96（组织核心影响力第 1 位）
  · 级联风险: 12 人连带流失 → 潜在总损失 ¥？ 
↓
王总截图拓扑图 + SHAP 归因面板 → 放入 Board 汇报 PPT
↓

周一 10:00 Board Meeting
王总投影 ONA 拓扑图：
  "各位，这是我们的组织网络拓扑。红点大小代表影响力。
  圆圈里有 12 条线从同一个节点散发——这个人如果离职，
  不是走一个人，是可能带崩 12 人的协作链。"

CFO 追问："你有数据吗？"
王总翻到下一页——ROI 测算表：
  "调薪 10%，投入 ¥15,600，净节约 ¥2,520。
  这是我们的测算模型，精确到每个节点的期望损失 ¥200,064。"

CEO 看向 CFO："给他批了。"
```

## 3. 角色对比矩阵

| 维度 | HRBP | CHO / HRD |
|------|------|-----------|
| **关注粒度** | 个体员工（谁要走、为什么、怎么留） | 组织级（人才风险敞口、系统稳定性） |
| **使用频率** | 日常（每日登录查看预警） | 周/月度（Board 汇报前深度使用） |
| **核心操作** | 悬停查看详情 → SHAP 分析 → 沙盘推演 → 导出 PDF | 全局拓扑浏览 → 风险敞口概览 → 截图/数据导出 |
| **成功标准** | "今天我拿着数据说服了业务主管" | "今天我用财务语言赢得了 Board 的预算批准" |
| **担忧的事** | 又有一个核心员工突然提离职 | 董事会说"HR 部门只是一个成本中心" |
| **ONA 切入点** | 个体诊断 + 干预工具 | 战略看板 + 风险仪表盘 |

## 5. 术语表（Bilingual）

| 英文术语 | 中文术语 | 说明 |
|---------|---------|------|
| Eigenvector Centrality (EC) | 特征向量中心度 | 衡量节点在组织网络中的核心影响力 |
| Hub Centrality | 枢纽中心度 | 节点的信息传递中介能力 |
| Cascade Failure | 级联失效 / 连带流失 | 关键节点离职引发的连锁反应 |
| Node Degree | 节点度 | 与该节点直接相连的邻居数 |
| SHAP Attribution | SHAP 归因 | 个体级特征贡献度分解 |
| Net Savings | 净节约金额 | 干预后预期损失 - 干预成本 |
| Retention Interview | 留任面谈 | 针对高危员工的结构化挽留沟通 |
| Pulse Survey | 脉冲问卷 / 高频调研 | 持续倾听平台的核心工具 |
| Passive ONA | 被动式组织网络分析 | 基于协作日志的非干预式分析 |
| White-Box Attribution | 白盒归因 | 可解释的个体级离职因子拆解 |
| Replacement Cost | 替换成本 | 招聘 + 培训 + 产能损失的总和 |
| Risk Exposure | 风险敞口 | 人才离职可能带来的财务损失总额 |
