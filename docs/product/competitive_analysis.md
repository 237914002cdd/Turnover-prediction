# 竞品分析报告：员工离职风险预测与管理平台

**文档状态**: v1.0 · 2026-06-11  
**分析对象**: Turnover ONA Platform vs Workday Peakon / Visier / Glint / Culture Amp  
**分析框架**: 差异化定位 + 深度对标 + SWOT  
**适用范围**: 内部战略规划、投资人沟通、竞标场景

## 1. 执行摘要：传统 HR Tech 的结构性缺陷

全球员工体验与人才保留市场在过去十年经历了从**年度敬业度调查**到**持续倾听平台**的演进。然而，这四家行业巨头（Workday Peakon、Visier、Glint、Culture Amp）的底层逻辑存在共同的根本性盲区——**它们都停留在"主动采集"范式**。

### 1.1 "调查陷阱"（The Survey Trap）

| 问题 | 说明 |
|------|------|
| **响应率衰减** | Pulse Survey 第一轮响应率通常 60-70%，到第 4 轮骤降至 30% 以下。员工产生"调查疲劳"。 |
| **主观偏差** | 员工自报数据受情绪周期、信任度、社会期望效应影响。声称"工作满意"的人仍可能在两周后提离职。 |
| **低频快照** | 即便最激情的持续倾听平台（Peakon）也只能做到周级采样。离职决策发生于调查间隙，系统无法感知。 |
| **组织盲区** | 传统工具只回答"谁不满意"（态度数据），回答不了"谁是信息枢纽"、"谁走了会带人走"（关系数据）。 |

### 1.2 ONA 的范式翻转

Turnover ONA 不依赖主动调查，而是通过**被动分析员工之间的协作图谱（Eigenvector Centrality）**，直接识别组织中的非正式权力中心。这不是对现有工具的补充——这是对人才风险识别方式的根本性重构。

| 维度 | 传统平台（Peakon / Glint / Culture Amp / Visier） | Turnover ONA |
|------|-------------------------------------------------|-------------|
| 数据来源 | 员工主动填写问卷 | 员工日常协作日志元数据 |
| 核心输出 | 敬业度分数 / eNPS / 满意度趋势 | 组织核心影响力 / 离职概率 / 级联风险 |
| 识别对象 | "谁不满意" | "谁是枢纽 + 谁可能走 + 走了影响多大" |
| 归因方式 | 统计回归（群体级） | SHAP 归因（个体级，可解释） |
| 干预建议 | "提升经理沟通质量"（通用） | "调薪 10% 可净节约 ¥2,520"（精确到元） |
| 数据时效性 | 调查周期（周/月） | 协作日志更新（T+1 天级） |

## 2. 竞品深度对标

### 2.1 Culture Amp & Glint：调查依赖型平台

**定位**: 员工敬业度与体验管理平台  
**共同弱点**: 重度依赖 Pulse Survey

| 对比维度 | Culture Amp / Glint | Turnover ONA |
|---------|---------------------|-------------|
| 数据采集 | 主动问卷（响应率逐月衰减） | 被动行为日志（无员工负担） |
| 核心指标 | eNPS / 敬业度分数 / 满意度 | 离职概率 / ONA 中心度 / 净节约金额 |
| 组织洞察 | 部门级分数对比 | 个体级枢纽识别（如申鹏程 EC=0.96） |
| 干预闭环 | 建议"改进经理反馈频率" | 精确"调薪 10% → 净省 ¥2,520" |
| 级联风险 | 不可见 | 潜在团队流失放大效应（12 人连带） |

**关键差距**: Culture Amp 和 Glint 能告诉 HR"哪个部门的敬业度在下降"——但它们无法告诉 HR"哪个基层员工是整个技术团队的非正式中枢，他走了会带走三个人。"

---

### 2.2 Visier：传统 HR 仪表盘

**定位**: 人力规划与分析平台  
**核心能力**: 人员流动率、招聘成本、人力资本 ROI 的历史报表

| 对比维度 | Visier | Turnover ONA |
|---------|--------|-------------|
| 时间视角 | **回顾性**（上季度离职率 12%） | **预测性**（该员工未来 6 月离职概率 82%） |
| 组织网络 | 不分析 | Eigenvector Centrality 全量计算 |
| 分析粒度 | 部门/团队聚合 | 个体级归因（8 项 SHAP 因子降序） |
| 财务模型 | 离职成本估算（静态公式） | ROI 沙盘推演（调薪滑块 → 实时 Net Savings） |
| 干预工具 | 看板 + 报表 | 状态机 + LLM 剧本 + PDF 留任建议书 |

**关键差距**: Visier 是优秀的"后视镜"——它把已发生的离职算得明明白白。但 HR 需要的不是已发生的事故报告，而是**还没发生的台风路径预测**。

---

### 2.3 Workday Peakon：持续倾听疲劳

**定位**: 员工生命周期体验平台  
**核心机制**: 持续 Pulse Survey + 里程碑节点调查

| 对比维度 | Workday Peakon | Turnover ONA |
|---------|--------------|-------------|
| 频率 | 每周/每双周推送调查 | 实时（协作日志 T+1 更新） |
| 员工体验 | **调查负担**：每轮 3-5 分钟，年累计 > 2 小时 | **零负担**：不需员工任何操作 |
| 数据客观性 | 主观自评（"你最近感觉如何？"） | 客观行为（加班时长、跨部门协作频次） |
| 分析深度 | 情绪指数趋势线 | SHAP 归因：薪资增幅 0% → 贡献离职概率 16.3% |
| 中国适配 | 中文适配有限，部署在海外 | 原生中文，39 字段中文化映射 |

**关键差距**: Peakon 在西方市场是领先产品，但其持续倾听模式依赖**员工自愿花时间填问卷**。中国职场环境中，调查响应率通常在 20-40%，数据代表性存疑。Turnover ONA 的**零负担被动采集**是本土化落地的结构性优势。

## 3. Turnover ONA 三大核心差异化支柱

### 3.1 支柱一：被动式组织网络分析（Passive ONA）

**问题传统工具解决不了**: 组织架构图只显示正式汇报关系，但真实的工作流中的信息传递、技术决策、跨部门协调绕过了正式层级。

**我们的方案**: 基于协作日志频次矩阵，计算每个员工的 Eigenvector Centrality（特征向量中心度），识别非正式权力中枢。

**案例锚点**: 申鹏程（P5，月薪 ¥13,000）
- 传统视角：基层执行者，可替代
- ONA 视角：EC=0.96，组织核心影响力极高，离职将引发 12 人级联震荡
- 管理标签：隐性组织章鱼 / 高负载错配

**竞争壁垒**: Workday 和 Visier 不具备也不测量 ONA 指标。要复制这一能力，需要：协作数据源接入 + 幂迭代算法 + 至少 6 个月的模型校准。

---

### 3.2 支柱二：白盒 SHAP 归因框架（White-Box Attribution）

**问题传统工具解决不了**: 大多数平台只能告诉你"离职率在上升"，不能告诉你"具体是什么驱动了这个人的离职决策"。

**我们的方案**: 对每位高危员工输出 8 项核心不满驱动因子，按 SHAP 贡献值降序排列，每个因子附带当前值和相关系数。

**案例锚点**: 李夏（P7，月薪 ¥32,000，离职概率 71.7%）

| 序号 | 驱动因子 | 当前值 | 相关系数 | 贡献占比 |
|------|---------|--------|---------|---------|
| 1 | 薪资增幅 | 3% | +0.198 | 16.3% |
| 2 | 任现职年限 | 6 年 | +0.177 | 14.1% |
| 3 | 月度加班超载 | 50h/月 | +0.165 | 13.0% |

**竞争壁垒**: Glint 和 Culture Amp 使用因子分析和回归，但输出是群体级（"研发部门的驱动因素是经理支持"）。我们做到了 **个体级归因**，每个员工看到的 Top 3 因子可能完全不同。

---

### 3.3 支柱三：财务 ROI 沙盘（Financial ROI Sandbox）

**问题传统工具解决不了**: 发现风险后，HR 拿着敬业度报告去找 CEO 批调薪预算，CEO 问"为什么是他？凭什么加这么多？"——HR 答不上来。

**我们的方案**: 在个体诊断页内置交互式 ROI 模拟器。HR 拖动调薪滑块，系统实时计算：

$$P_{\text{new}} = P_{\text{base}} \times e^{-\alpha \cdot X}$$

$$\text{Net Savings} = \text{Pre\_Cost} - \text{Post\_Cost} - \text{Invest\_Cost}$$

**教科书级演示数据**（申鹏程 10% 调薪模拟）：

| 指标 | 调薪前 | 调薪后 |
|------|--------|--------|
| 月薪 | ¥13,000 | ¥14,300 |
| 离职概率 | 82% | ~20% |
| 预期离职花费 | ¥20,000 | ¥1,880 |
| 企业投入 | — | ¥15,600 |
| **净节约金额** | — | **¥2,520** |

**竞争壁垒**: Visier 有离职成本估算，但那是静态公式，不支持 HR 现场拖拽调整参数、实时看到数字变化。我们的沙盘是**交互式决策工具**，不是报表。

## 4. SWOT 分析

### 4.1 优势（Strengths）

| 优势 | 说明 |
|------|------|
| **ONA 护城河** | 竞品不测量组织网络中心度，这是 6-12 个月的追赶壁垒 |
| **个体级归因精度** | SHAP 框架输出 8 项因子降序，竞品止步于群体级分析 |
| **财务语言闭环** | ROI 沙盘把 HR 需求翻译成 CFO 听得懂的 Net Savings |
| **零负担数据采集** | 不需员工填问卷，适合中国职场低响应率环境 |
| **全栈闭环** | 数据导入 → 诊断 → 沙盘推演 → PDF 导出，15 分钟内完成 |

### 4.2 劣势（Weaknesses）

| 劣势 | 说明 | 缓解策略 |
|------|------|---------|
| **品牌认知度** | 零品牌基础，对标 Workday 等上市公司 | 以 ROI 案例驱动口碑 |
| **数据源依赖** | ONA 需要协作日志元数据，中小客户可能缺乏 | MVP 阶段提供样本数据预演 |
| **销售团队** | 无 ToB 销售体系 | 先做 PLG（产品驱动增长） |
| **产品成熟度** | v0.3.0 尚缺认证/权限/部署脚本 | 路线图中已列为 NEXT 阶段 |

### 4.3 机会（Opportunities）

| 机会 | 说明 |
|------|------|
| **中国 HR Tech 蓝海** | 中国 HCM 市场 SAP/Oracle 份额低，本土产品无 ONA 功能 |
| **AI+HR 政策红利** | 多地政府推动企业数字化转型，HR 科技预算增长 |
| **Workday 退出中国** | 部分外企客户正在寻找替代方案 |
| **远程办公常态化** | 协作数据更加丰富，ONA 分析价值放大 |
| **CHO 向 CFO 汇报趋势** | 越来越多 CHO 需要用财务语言与 CEO 沟通，ROI 沙盘精准击中需求 |

### 4.4 威胁（Threats）

| 威胁 | 说明 | 应对 |
|------|------|------|
| **Workday 收购整合** | 如果 Workday 收购一家 ONA 初创，市场规则改变 | 加快地基建设，锁定首批客户 |
| **数据隐私监管** | 协作日志分析可能引发员工隐私质疑 | 物理层阻断聊天文本，仅存元数据 |
| **大厂入局** | 飞书/钉钉/企业微信可能内置类似功能 | 走 OpenAPI 嵌入路线，做生态而非竞对 |
| **经济下行** | HR Tech 预算可能被削减 | ROI 测算直接量化省钱效益，帮客户算账 |

### 定位声明

> Turnover ONA 不竞争"更好的问卷平台"。它开辟了一个新品类——**基于组织网络分析的非正式风险识别系统**。在这个品类中，目前没有可对标竞品。

现有竞品覆盖了"敬业度测量"和"人力报表"两个象限。Turnover ONA 覆盖的是**第三个象限——非正式组织网络的被动式风险扫描**。这不是功能层面的竞争，是品类层面的错位。
