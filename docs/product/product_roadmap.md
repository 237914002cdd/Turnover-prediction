## NEXT — 下一阶段（2026 Q3 · 1-3 个月）

### 🔴 P0 — 生产刚需
| 项目 | 预估 | 说明 |
|------|------|------|
| **用户认证与 RBAC 鉴权** | 2-3 周 | HRBP / HRD / Admin 三级角色，JWT 登录，薪资字段按角色脱敏 |
| **Docker 容器化部署** | 1 周 | Dockerfile + docker-compose + nginx 反向代理 + env 配置 |

### 🟡 P1 — 体验增强

|------|------|------|

| **响应式布局适配** | 1 周 | 适配 1366×768 笔记本分辨率，下钻页双栏折叠 |
| **数据仪表盘定制化** | 2 周 | 用户可按需拖拽配置 Dashboard 组件位置 |

### 🟢 P2 — 算法迭代
| 项目 | 预估 | 说明 |
|------|------|------|
| **真实 ML 模型接入** | 3-4 周 | 当前离职概率/归因系数为硬编码，替换为由 Python 训练的随机森林模型实时推理 |
| **全量 ONA 重收敛流水线** | 1 周 | 定时任务每月自动执行 `build_ona_feature_engineering_pipeline.py` |

## LATER — 远期规划（2027 H1+）

### 智能化
| 项目 | 说明 |
|------|------|
| **Org-LLM Copilot** | 自然语言对话式人才诊断："帮我看看哪个部门的离职风险最高"→系统自动查询并展示 |
| **离职预测时间窗口** | 从'是否离职'二分类升级为'未来 3/6/12 个月内离职概率' |
| **干预推荐引擎** | 基于历史干预结果（feedback_history 表），自动推荐最优留任手段 |

### 规模化
| 项目 | 说明 |
|------|------|
| **跨行业离职基准库** | 聚合各行业离职率数据，形成行业对标 benchmark |
| **部门级流失漏斗** | 从入职→转正→稳定→高危→离职的完整周期追踪 |
| **多租户 SaaS 架构** | 支持多家企业独立数据隔离 |

### 生态
| 项目 | 说明 |
|------|------|
| **飞书/钉钉/企业微信集成** | 单点登录 + 数据同步 + 消息推送三端打通 |
| **Workday/SAP SuccessFactors 适配器** | 标准 HRIS 数据源适配器 |
